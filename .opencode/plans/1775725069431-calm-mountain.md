# Plan

## Recommendation

- 当前不需要改代码来支持 Kubernetes PVC。
- 继续保留镜像约定：SQLite 数据库默认位于 `/data/auth.sqlite`。
- 在 Kubernetes 中，最简单且最兼容的方式是把 PVC 挂载到 `/data`。

## Meaning Of `VOLUME ["/data"]`

- 它表示镜像把 `/data` 视为持久化数据目录。
- 当前默认启动命令会把实例数据库路径写死为 `/data/auth.sqlite`。
- 所以容器真正要持久化的是这个 SQLite 文件，而不是 `VOLUME` 指令本身。

## Kubernetes PVC Approach

- 在 Pod/Deployment 里声明一个 PVC。
- 把该 PVC 挂载到容器的 `/data`。
- 保持当前默认命令不变，或继续传 `--issuer ...` 等额外参数。

示意：

```yaml
volumeMounts:
  - name: auth-mini-data
    mountPath: /data

volumes:
  - name: auth-mini-data
    persistentVolumeClaim:
      claimName: auth-mini-data
```

## Alternative

- 如果将来你想把 PVC 挂到别的路径，比如 `/var/lib/auth-mini`，也可以。
- 但那样就要把启动参数里的数据库路径从 `/data/auth.sqlite` 改成新的挂载路径。
