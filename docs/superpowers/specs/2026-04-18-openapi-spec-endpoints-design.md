# OpenAPI 文档端点设计

## 背景/目标

- 当前仓库已有 `openapi.yaml` 作为 HTTP API 合同来源，但运行中的服务尚未直接暴露该合同文件。
- 需要为服务新增可读取的 OpenAPI 文档端点，方便调试、集成和外部工具直接获取合同内容。
- 本轮目标是在服务启动时一次性读取打包后的 `openapi.yaml`，同时缓存原始 YAML 字符串和基于该 YAML 解析得到的 JSON 对象，并分别通过 `GET /openapi.yaml` 与 `GET /openapi.json` 对外返回。
- `openapi.yaml` 是唯一事实来源；JSON 仅是运行时派生缓存，不写回磁盘，也不作为新的源文件维护。

## 范围

- 启动阶段从发布产物内可访问的位置读取 `openapi.yaml`。
- 启动阶段解析 YAML，并把原始字符串与解析后的 JSON 对象缓存到内存中。
- 新增 `GET /openapi.yaml`，返回启动时读取的原始 YAML 内容。
- 新增 `GET /openapi.json`，返回同一份 YAML 解析后的 JSON 内容。
- 确保 `openapi.yaml` 被包含进最终发布包或构建产物，运行时代码从打包后文件位置读取，而不是依赖源码 checkout 的当前工作目录。
- 更新 OpenAPI 合同与契约测试，使新增端点与实际行为一致。

## 非目标

- 不新增第二份持久化的 `openapi.json` 文件。
- 不在请求阶段重复读取或重复解析 `openapi.yaml`。
- 不引入运行时热重载、文件监听或动态刷新缓存能力。
- 不改变现有业务 API 的语义、认证或路由结构，除新增两个文档端点外不扩展其它 HTTP 能力。
- 不把 OpenAPI 文件改成从远程地址、环境变量内容或数据库读取。

## 设计方案

### 启动缓存模型

- 服务启动时定位打包后可访问的 `openapi.yaml` 文件，并同步完成读取。
- 读取成功后，保留两份内存数据：
  - 原始 YAML 字符串，用于 `GET /openapi.yaml`。
  - 基于该 YAML 解析出的 JSON 对象，用于 `GET /openapi.json`。
- 两个端点均只读取启动期缓存，不在请求阶段触发文件 I/O 或 YAML 解析。

### 路由行为

- `GET /openapi.yaml`
  - 返回启动期缓存的原始 YAML 字符串。
  - 响应内容应保持与打包文件内容一致。
- `GET /openapi.json`
  - 返回由同一份 YAML 解析得到的 JSON 对象。
  - 返回值仅来自启动期缓存，保证与 `GET /openapi.yaml` 指向同一事实来源。

### 单一事实来源

- 仓库内只维护 `openapi.yaml` 这一份 OpenAPI 合同文件。
- JSON 表示是运行时派生结果，仅存在于内存中，不落盘、不发布为单独源文件、不允许人工维护。
- 后续合同更新流程仍以修改 `openapi.yaml` 为起点，再同步服务实现、测试与文档端点验证。

## 错误处理

- 如果启动时无法读取 `openapi.yaml`，服务必须直接启动失败，禁止带缺失文档端点继续运行。
- 如果启动时 YAML 解析失败，服务必须直接启动失败，禁止降级为只提供 YAML 或返回空 JSON。
- 请求阶段不额外设计新的恢复逻辑；因为端点依赖的是已成功初始化的内存缓存，启动失败已在前置阶段阻断。

## 发布与打包要求

- `openapi.yaml` 必须进入最终 npm 包或对应构建产物，不能只存在于源码目录。
- 运行时代码必须基于已发布产物中的文件路径读取 `openapi.yaml`，不能假设进程当前工作目录是仓库根目录。
- 打包验证需要覆盖发布结果中确实包含 `openapi.yaml`，并且运行时代码可从该位置完成启动读取。

## 测试与验证

- 新增端点测试，覆盖 `GET /openapi.yaml` 返回原始 YAML，`GET /openapi.json` 返回解析后的 JSON。
- 新增启动失败测试，覆盖文件缺失与 YAML 解析失败两类场景。
- 新增打包验证，确认发布产物包含 `openapi.yaml`，且运行时读取路径在打包后仍然成立。
- 更新 OpenAPI 合同与相关契约测试，确保新增端点被文档化并与实际响应一致。
- 实现阶段采用先补测试再补实现的方式推进。

## 影响文件

- `openapi.yaml`
  - 更新合同，纳入 `GET /openapi.yaml` 与 `GET /openapi.json`。
- 服务启动与应用装配相关文件
  - 新增 OpenAPI 文件读取、解析与缓存注入逻辑。
- 服务路由相关文件
  - 新增两个文档端点并消费启动期缓存。
- 打包配置相关文件
  - 确保 `openapi.yaml` 被包含进发布包或构建产物。
- 测试文件
  - 新增端点测试、启动失败测试、打包验证与契约测试更新。
