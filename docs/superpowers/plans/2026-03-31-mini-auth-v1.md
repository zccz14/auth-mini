# auth-mini v1 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first working release of `auth-mini`: a CLI-only Hono auth server with email OTP login, Ed25519 JWT/JWKS, refresh sessions, SMTP-backed delivery, and WebAuthn passkey registration/authentication.

**Architecture:** The project is a TypeScript Node.js CLI application. `cac` handles command routing; `zod` validates command input and runtime config; `better-sqlite3` plus handwritten SQL own persistence; Hono exposes HTTP routes; domain modules split email auth, sessions, WebAuthn, and JWKS. Integration tests drive the main flows against a real SQLite database, local mock SMTP server, and real server-side WebAuthn verification.

**Tech Stack:** TypeScript, Node.js, Hono, cac, zod, better-sqlite3, Vitest, ESLint, Prettier, Husky, lint-staged, SMTP, WebAuthn, Ed25519/EdDSA JWT/JWKS

## Implementation Contract

### CLI contract

- `auth-mini create <dbPath> [--smtp-config <jsonFile>]`
- `auth-mini start <dbPath> [--host <host>] [--port <port>] --issuer <url> --rp-id <rpId> --origin <origin> [--origin <origin> ...]`
- `auth-mini rotate-jwks <dbPath>`

Runtime config source for v1:

- source of truth is CLI flags only
- `--host` default = `127.0.0.1`
- `--port` default = `7777`
- `--issuer` required
- `--rp-id` required
- `--origin` required and repeatable; parse as a string array in declaration order
- invalid config prints a compact error to stderr and exits with code `1`

### HTTP/token contract

- access token transport: `Authorization: Bearer <jwt>`
- refresh token transport: JSON body `{ "refresh_token": "..." }`
- v1 does not use cookies

Standard token success response:

```json
{
  "access_token": "<jwt>",
  "token_type": "Bearer",
  "expires_in": 900,
  "refresh_token": "<opaque-token>"
}
```

### Core endpoint contract

`POST /email/start`

Request:

```json
{ "email": "user@example.com" }
```

Success response:

```json
{ "ok": true }
```

Known failure:

- `503 smtp_not_configured`

`POST /email/verify`

Request:

```json
{ "email": "user@example.com", "code": "123456" }
```

Success response: standard token success response

`GET /me`

Success response:

```json
{
  "user_id": "<uuid>",
  "email": "user@example.com",
  "webauthn_credentials": [
    {
      "id": "<db-id>",
      "credential_id": "<base64url>",
      "transports": ["internal"],
      "created_at": "<iso-datetime>"
    }
  ],
  "active_sessions": [
    {
      "id": "<session-id>",
      "created_at": "<iso-datetime>",
      "expires_at": "<iso-datetime>"
    }
  ]
}
```

`active_sessions` includes only sessions where `revoked_at IS NULL` and `expires_at > now()`.

Known failures:

- `401 invalid_access_token`

`POST /session/refresh`

Request:

```json
{ "refresh_token": "<opaque-token>" }
```

Success response: standard token success response

Known failures:

- `401 invalid_refresh_token`

`POST /session/logout`

Success response:

```json
{ "ok": true }
```

`GET /jwks`

Success response:

```json
{ "keys": [] }
```

`POST /webauthn/register/options`

Success response:

```json
{
  "request_id": "<uuid>",
  "publicKey": {
    "challenge": "<base64url>",
    "rp": { "name": "auth-mini", "id": "example.com" },
    "user": {
      "id": "<bytes>",
      "name": "user@example.com",
      "displayName": "user@example.com"
    },
    "pubKeyCredParams": [],
    "timeout": 300000,
    "authenticatorSelection": {
      "residentKey": "required",
      "userVerification": "preferred"
    }
  }
}
```

`POST /webauthn/register/verify`

Request:

```json
{
  "request_id": "<uuid>",
  "credential": {
    "id": "<credential-id>",
    "rawId": "<base64url>",
    "response": {},
    "type": "public-key"
  }
}
```

Success response:

```json
{ "ok": true }
```

Known failures:

- `400 invalid_webauthn_registration`
- `409 duplicate_credential`

`POST /webauthn/authenticate/options`

Success response:

```json
{
  "request_id": "<uuid>",
  "publicKey": {
    "challenge": "<base64url>",
    "rpId": "example.com",
    "timeout": 300000,
    "userVerification": "preferred"
  }
}
```

`POST /webauthn/authenticate/verify`

Request:

```json
{
  "request_id": "<uuid>",
  "credential": {
    "id": "<credential-id>",
    "rawId": "<base64url>",
    "response": {},
    "type": "public-key"
  }
}
```

Success response: standard token success response

Known failures:

- `400 invalid_webauthn_authentication`

`DELETE /webauthn/credentials/:id`

- `:id` is the database row id returned by `/me`

Success response:
